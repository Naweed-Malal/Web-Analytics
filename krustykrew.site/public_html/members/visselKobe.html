Vissel Kobe Project

Over the summer of 2021, I was given the opportunity to travel to Tokyo, Japan where I had the opportunity to work for an internship with Rakuten in the Sports Service Development Department. My job was to create the voiceover implementation for the Vissel Kobe app.


Objective

My goal was to create the voiceover functionality for the iOS app, Vissel Kobe.


The Vissel Kobe app was created for the Vissel Kobe soccer team in Japan. Vissel Kobe provides users with easy access to team information, schedules, standings, and more. You can also purchase match tickets using Vissel Kobe.


In iOS and other apple systems, there are four areas of focus: motor, cognitive, vision, and hearing. Accessibility is important to improve your products' inclusivity and performance amongst all types of people. It should be easy and intuitive to navigate through your app. VoiceOver is a gesture-based screen reader that enables people to experience the interface on their devices without having to see the screen. It allows people who are blind or have low vision to use their device with the auditory feedback voiceover provides.


Personal Challenges

Prior to this internship, I had never used Swift before. I had to learn how to use Swift and how voiceover should work before starting the project. Since my internship was during COVID-19, I was the only intern working on this project. There was also a language barrier between me and the rest of the team I worked with.


Results

Most of the elements on the app were missing accessibility labels, traits, and/or hints. Some elements would have default values set for these settings which would lead to voiceover reading any text it could find, which could be confusing to users without background information. Many elements within containers such as table or collection views would be combined as one big element and voiceover would miss all of that information. These problems could be easily fixed by adding the necessary accessibility variables.


The app had several features that needed to be altered when in voiceover mode. On the homepage, the Vissel Kobe app has an element that automatically scrolls. Voiceover could not recognize this as an accessibility element.

Voiceover did not function properly when trying to swipe through the pages and could not handle the automatic scrolling


To fix this issue, I set the automatic scrolling off when voiceover is running. Then, I set it as an adjustable element so that you could three finger swipe or swipe up and down with one finger to change the page.


The automatic scrolling is off since voiceover is on. Voiceover now functions properly.


Voiceover was now recognizing these elements as three different pages, but the accessibility label was giving the same values for each page. This had to be fixed because each page led to a different link. This issue required looking into the API and setting a new variable that had the proper voiceover description. Then, I could use this variable and retrieve its string value to set as the accessibility label. I added a UIAccessibilityPostNotification to read the page number after a user swipes with voiceover on.


Despite these challenges, I was able to implement voiceover in the Vissel Kobe app. The team is currently working on multiple testing cycles prior to releasing this development.
